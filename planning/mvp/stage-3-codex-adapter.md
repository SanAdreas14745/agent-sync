# Этап 3. Codex Adapter

Статус: planning-документ.

Этот документ фиксирует рабочий план и критерии готовности этапа MVP.

## Цель

Преобразовать результат Composer-а в структуру файлов, которую Codex
сможет использовать в локальном проекте.

## Вход

Codex adapter получает:

``` ts
interface CodexAdapterInput {
  resolveResult: ResolveResult;
  output: {
    entry: string;
    generatedDir: string;
  };
}
```

## Выход

Adapter генерирует:

``` text
AGENTS.md
.agents/generated/active-rules.md
.agents/generated/rule-scope.md
.agents/generated/skill-index.json
.agents/generated/rules/*.md
```

## Принцип

`AGENTS.md` должен быть коротким входным файлом. Он не должен превращаться
в огромный архив всех правил.

`active-rules.md` остается коротким bootstrap-файлом. Основной объем
правил размещается в `.agents/generated/rules/*.md`, а карта применимости
лежит в `.agents/generated/rule-scope.md`. Optional и reference материалы
попадают в `.agents/generated/skill-index.json`.

## AGENTS.md

Пример:

``` md
# Agent Instructions

This repository uses centralized AI skills.

Always follow:
- .agents/generated/active-rules.md

Available optional skills and references:
- .agents/generated/skill-index.json

Generated files. Do not edit manually.
Run `ai-skills sync` to update them.
```

Для MVP `AGENTS.md` можно полностью перезаписывать.

Позже можно добавить режим с managed-блоком, чтобы сохранить ручные
локальные инструкции.

## active-rules.md

Пример:

``` md
<!--
Generated by ai-skills
Generated at: 2026-07-07T00:00:00.000Z
Project: statistics
Agent: codex
Included skills:
- company.common@1
- frontend.common@2
- project.statistics.api@1
Checksum: 4e2a...
-->

# Active AI Rules

## company.common

...

## frontend.common

...

## project.statistics.api

...
```

## skill-index.json

Пример:

``` json
{
  "generatedAt": "2026-07-07T00:00:00.000Z",
  "agent": "codex",
  "project": "statistics",
  "included": [
    {
      "id": "frontend.common",
      "title": "Frontend Common Rules",
      "version": 2,
      "loadMode": "project"
    }
  ],
  "available": [
    {
      "id": "reference.project-architecture",
      "title": "Project Architecture",
      "summary": "Architecture reference for the project",
      "loadMode": "reference",
      "sourceFile": "references/project-architecture.md"
    }
  ],
  "warnings": []
}
```

## Manifest и checksum

Manifest нужен для проверяемости:

-   какие правила вошли;
-   какие версии вошли;
-   когда сгенерировано;
-   для какого проекта и агента;
-   изменился ли результат после следующего `sync`.

Checksum можно считать по нормализованному содержимому generated-файлов
или по списку `id@version + body`.

Для MVP достаточно стабильного checksum, который меняется при изменении
active-набора.

## Правила записи файлов

Adapter должен:

-   создавать generated-директорию, если ее нет;
-   перезаписывать managed-файлы;
-   не удалять файлы вне managed-набора;
-   завершаться ошибкой, если output path выходит за пределы проекта;
-   использовать стабильные переводы строк;
-   формировать одинаковый результат при одинаковом input.

## Managed files

MVP managed files:

``` text
AGENTS.md
.agents/generated/active-rules.md
.agents/generated/skill-index.json
```

Если в будущем появятся ручные секции, нужно будет использовать managed
markers:

``` md
<!-- ai-skills:start -->
...
<!-- ai-skills:end -->
```

В MVP это не требуется.

## Тесты

Минимальные тесты:

-   генерирует `AGENTS.md`;
-   генерирует `active-rules.md`;
-   генерирует `skill-index.json`;
-   добавляет manifest;
-   включает список `id@version`;
-   сохраняет стабильный порядок правил;
-   повторный запуск не меняет файлы при том же input;
-   adapter не пишет за пределы output-директории.

## Критерии готовности

-   generated-файлы читаемы человеком;
-   Codex получает ссылку на active rules через `AGENTS.md`;
-   generated-файлы содержат manifest;
-   output стабилен;
-   adapter не зависит от registry reader;
-   adapter принимает ResolveResult и использует стандартные output paths
    для Codex.

## Выход этапа

Файловая структура для Codex, пригодная для проверки в реальном проекте.
