# YouTrack Development Backlog Task

## Назначение

Превращать сырой запрос пользователя в понятное тело задачи для backlog
разработки в YouTrack: читаемое для постановщика, исполнителя, QA и агента,
который может продолжить работу по задаче.

## Главный Принцип

Не заполняй задачу догадками. Сначала собери достаточный контекст из запроса и
релевантных проектных файлов, затем либо задай уточняющие вопросы, либо
составь готовое тело задачи.

## Workflow

1. Убедись, что запрос относится к задаче backlog разработки. Если тип задачи
   другой, не применяй этот шаблон без явного согласия пользователя.
2. Изучи исходный запрос и выдели проблему, ожидаемую пользу, затронутые
   сценарии, ограничения и неизвестные.
3. Найди и прочитай релевантные проектные файлы: компоненты, сервисы,
   контракты, state, routing, тесты, конфиги или документацию, которые нужны
   для точной постановки.
4. Если без ответа нельзя корректно сформулировать задачу, задай
   уточняющие вопросы отдельным сообщением и не создавай черновик как факт.
5. Если суть ясна, составь тело задачи по структуре из
   [references/STRUCTURE.md](references/STRUCTURE.md).
6. Прикладывай кодовые фрагменты только когда они помогают исполнителю быстро
   найти место изменения или понять контракт. Не вставляй большие файлы целиком.
7. Используй временную заглушку интеграции: создание задач в YouTrack будет
   реализовано позже через YouTrack MCP server.
8. Пока MCP-подключение не реализовано, не создавай задачу самостоятельно и
   пришли готовое тело задачи в чат.

## Чек-Лист

- Задача относится именно к backlog разработки.
- В задаче есть user story, формальное описание, детали реализации и ожидаемый
  результат.
- Неизвестные не замаскированы под факты.
- QA получает проверяемый результат или чеклист.
- Интеграция с YouTrack обозначена как будущая MCP-возможность.
- Задача в YouTrack не создана напрямую до появления MCP-подключения.
