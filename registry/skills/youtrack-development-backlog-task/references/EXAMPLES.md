# Примеры YouTrack Development Backlog Task

## Хороший Мини-Кейс

Сырой запрос:

``` text
Нужно добавить сохранение выбранного периода в отчете продаж, сейчас после
обновления страницы фильтр сбрасывается.
```

Фрагмент хорошего тела задачи:

``` markdown
## User story

Я, как пользователь отчета продаж, хочу сохранять выбранный период после
обновления страницы, чтобы продолжать анализ без повторной настройки фильтра.

## Описание задачи

Для того, чтобы пользователи могли возвращаться к выбранному периоду после
перезагрузки страницы, нужно сохранять значение фильтра периода и
восстанавливать его при повторном открытии отчета продаж.

## Детали реализации

- [ ] проверить текущую реализацию фильтра периода в модуле отчета продаж;
- [ ] определить, где в проекте уже сохраняются пользовательские фильтры;
- [ ] восстановить выбранный период при инициализации страницы;
- [ ] не менять поведение остальных фильтров без отдельного требования.

## Ожидаемый результат

После выбора периода и обновления страницы отчет продаж открывается с тем же
периодом.

Скриншоты не приложены.

Комментарий для QA:

- [ ] открыть отчет продаж;
- [ ] выбрать период;
- [ ] обновить страницу;
- [ ] проверить, что выбранный период сохранился;
- [ ] проверить, что остальные фильтры работают как раньше.
```

## Пример С Кодовым Фрагментом

Добавляй короткий фрагмент, если он помогает локализовать изменение:

```` markdown
В текущей реализации значение сбрасывается при инициализации:

```ts
this.form.patchValue({ period: DEFAULT_PERIOD });
```

Проверь, можно ли заменить безусловную установку дефолта восстановлением
сохраненного значения.
````

Не вставляй весь компонент, сервис или reducer целиком.

## Когда Нужно Спросить

Сырой запрос:

``` text
Сделай нормальное поведение кнопки подтверждения.
```

Нужно задать вопросы:

``` markdown
Мне нужно уточнить несколько вещей, чтобы корректно составить задачу:

1. На каком экране находится кнопка подтверждения?
2. Что сейчас считается неправильным поведением?
3. Какой результат должен проверить QA?
```

## Типовые Ошибки

- Пытаться создать задачу в YouTrack до появления YouTrack MCP-подключения.
- Проверять локальные способы доступа вместо использования временной заглушки
  MCP-интеграции.
- Придумать бизнес-роль, права доступа или acceptance criteria без опоры на
  запрос и проектные файлы.
- Смешать уточняющие вопросы и якобы готовую постановку.
- Перегрузить задачу внутренними рассуждениями агента.
- Добавить технические шаги, которые навязывают архитектуру без необходимости.
